# Data Privacy Frameworks Reference

## Major Frameworks Comparison

| Framework | Region | Effective | Key Requirements | Penalty |
|-----------|--------|-----------|------------------|---------|
| GDPR | EU/EEA | 2018 | Consent, rights, DPO, breach notification | Up to 4% global revenue |
| CCPA/CPRA | California | 2020/2023 | Disclosure, opt-out, deletion | $7,500 per violation |
| LGPD | Brazil | 2020 | Similar to GDPR, local DPO | 2% revenue, max R$50M |
| PIPL | China | 2021 | Consent, localization, cross-border | Up to 5% revenue |
| POPIA | South Africa | 2021 | Processing principles, consent | Up to R10M |
| PDPA | Singapore | 2012 (amended) | Consent, purpose limitation | Up to S$1M |

## GDPR Deep Dive

### Core Principles (Article 5)

1. **Lawfulness, fairness, transparency**: Legal basis, clear communication
2. **Purpose limitation**: Collect only for specified purposes
3. **Data minimization**: Only necessary data
4. **Accuracy**: Keep data accurate and current
5. **Storage limitation**: Don't keep longer than needed
6. **Integrity and confidentiality**: Appropriate security
7. **Accountability**: Demonstrate compliance

### Legal Bases for Processing

| Basis | Use Case | Documentation |
|-------|----------|---------------|
| Consent | Marketing, cookies | Explicit, withdrawable |
| Contract | Service delivery | Contract terms |
| Legal obligation | Tax records | Legal requirement |
| Vital interests | Emergency medical | Rare, specific situations |
| Public task | Government functions | Official authority |
| Legitimate interest | Fraud prevention | Balancing test required |

### Data Subject Rights

| Right | Description | Response Time |
|-------|-------------|---------------|
| Access | Copy of their data | 1 month |
| Rectification | Correct inaccurate data | 1 month |
| Erasure | Delete their data | 1 month |
| Portability | Machine-readable export | 1 month |
| Restriction | Limit processing | 1 month |
| Object | Stop certain processing | 1 month |
| Automated decisions | Human review | 1 month |

### Data Protection Officer (DPO)

**Required When**:
- Public authority
- Large-scale systematic monitoring
- Large-scale special category data

**Responsibilities**:
- Advise on compliance
- Monitor compliance
- Contact point for authorities
- DPIA oversight

### Cross-Border Transfers

**Approved Mechanisms**:
1. Adequacy decision (limited countries)
2. Standard Contractual Clauses (SCCs)
3. Binding Corporate Rules (BCRs)
4. Explicit consent (limited use)
5. Derogations (specific situations)

## CCPA/CPRA Overview

### Consumer Rights

| Right | CCPA | CPRA Addition |
|-------|------|---------------|
| Know | ✓ | Enhanced |
| Delete | ✓ | Enhanced |
| Opt-out of sale | ✓ | + Sharing |
| Non-discrimination | ✓ | ✓ |
| Correct | - | ✓ |
| Limit sensitive data | - | ✓ |
| Access automated decisions | - | ✓ |

### Business Obligations

- **Privacy Policy**: Disclose data practices
- **Do Not Sell Link**: Homepage requirement
- **Opt-out mechanism**: Honor requests
- **Data inventory**: Know what you have
- **Vendor contracts**: Appropriate terms

### Thresholds

Applies to businesses that:
- $25M+ gross revenue, OR
- Buy/sell 100K+ consumers' data, OR
- 50%+ revenue from selling data

## Privacy Framework Compliance Checklist

### Data Mapping
- [ ] Inventory all personal data collected
- [ ] Document data flows
- [ ] Identify legal basis for each processing activity
- [ ] Map third-party data sharing
- [ ] Document retention periods

### Consent Management
- [ ] Implement consent mechanism
- [ ] Track consent preferences
- [ ] Enable consent withdrawal
- [ ] Age verification (if applicable)
- [ ] Cookie consent (GDPR)

### Data Subject Rights
- [ ] Process for receiving requests
- [ ] Identity verification procedure
- [ ] Response templates
- [ ] Tracking/audit system
- [ ] Timeline monitoring

### Security
- [ ] Encryption at rest
- [ ] Encryption in transit
- [ ] Access controls
- [ ] Audit logging
- [ ] Incident response plan

### Documentation
- [ ] Privacy policy (public)
- [ ] Internal privacy procedures
- [ ] Data processing agreements
- [ ] DPIA for high-risk processing
- [ ] Records of processing activities

### Training
- [ ] Employee privacy training
- [ ] Role-specific training
- [ ] Annual refresher
- [ ] New hire onboarding

## Regional Quick Reference

### EU/EEA
- **Framework**: GDPR
- **Authority**: National DPAs
- **Key**: Consent, rights, DPO, SCCs

### United States
- **Federal**: Sector-specific (HIPAA, COPPA, GLBA)
- **States**: CA, VA, CO, CT, UT (comprehensive)
- **Key**: Opt-out, disclosure, Do Not Sell

### APAC
- **Strict**: China (PIPL), Singapore (PDPA)
- **Developing**: India (DPDP), Indonesia, Vietnam
- **Key**: Localization, consent, cross-border

### LATAM
- **Key Law**: Brazil (LGPD), Argentina
- **Trend**: GDPR-style adoption
- **Key**: Consent, DPO, local presence
